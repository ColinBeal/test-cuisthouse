Guidelines

Why guidelines?

These guidelines are a set of components and good practices, these principles are made to be reusable throughout the Cuisthouse application, their usage simplify the designing process of new interfaces and also make their integration easier for the dev team.

Colors

Main color

$main-color

The main color of the application is orange (#F5A22F), you can use it in scss by calling the $main-color variable.
It's widely used in every of the other components primary states.

Accent color

$accent-color

The accent color of the application is white (#FFFFFF), you can use it in scss by calling the $accent-color variable.
It's used on the secondary states of components and in order to contrast on main color background.

Body color

$body-color

The body color of the application is a really dark grey (#242424), you can use it in scss by calling the $body-color variable.
It's used for all the text in the application as default, differents variants with an opacity tuned downed are used for long texts.

Breadcrumb

Favorites
Apple Frangipane Tart
        
          <div class="breadcrumb">
            <div class="item" routerLink="/favorites">Favorites</div>
            <img src="https://raw.githubusercontent.com/ColinBeal/test-cuisthouse/master/src/assets/img/chevron-right.png" alt="chevron right">
            <div class="active item">Apple Frangipane Tart</div>
          </div>

The breadrumb components is used to help the user know where he is in the application while he is navigating.
It's defined by the class breadcrumb and nest item class div each of them separated by chevron icons
We do not use it on the main page, but for every "/" in the route path you should add an item in the breadcrumb. The current state is defined by an active item. Items that are not active should have a routerLink so that the user can easily navigate to the parent elements.
It should always be used at the beginning of the view so it appears just after the nav-bar in dom.

Button

Button

See my favorites
        
          <div class="button">
            <div routerLink="/favorites">See my favorites</div>
          </div>

The default button is used on each view as the main action. It's defined with the class button.

Accent Button

See my favorites
        
          <div class="accent button">
            <div routerLink="/favorites">See my favorites</div>
          </div>

The accent button is a variant of button, it is used on each view as a secondary action. It's defined by adding the class accent. Note that it can only be used on a main-color background.

Actions Button

            <div>Add to my favorites</div>

          <div class="red button">

The action button is a variant of button, he is used for a specific action, the red one is used only for deleting actions and the blue one for creation actions. They are defined by adding the class red or blue.

Card

Single card

        
          <app-card [card]="card"></app-card>

The card component is used to display a quick abstract of a recipe to the user.
The card item is an angular component defined in the dom as app-card. It takes a card object as an input (defined in card.model.ts).

Cards list

        
          <div class="cards-list-container">
            <div class="cards-list">
              <app-card *ngFor="let card of cards" [card]="card"></app-card>
            </div>
          </div>

The cards list component is used to display list of cards, it's the most common way of displaying recipes cards.
The cards-list class is nested in a cards-list-container that adds the orange background wich is used by default on most pages. You can remove the orange background by adding the class no-bg after the cards-list-container.

Difficulty Rating

Default Difficulty Rating

        
          <difficulty-rating [rating]="3"></difficulty-rating>

The difficulty rating component is used to give the information about the difficulty of the recipe, the default use is in other styled component.
The difficulty rating item is an angular component defined in the dom as difficulty-rating. It takes a rating; a number between 1 and 5 as an input.

Big Difficulty Rating

        
          <difficulty-rating class="big" [rating]="3"></difficulty-rating>

The big difficulty rating is a variant used in heading to give the same information but with bigger forks.
To be used the class big should be added to the element difficulty-rating

Field

Input field

        
          <input type="text" class="field" placeholder="search for a recipe">

The input field component permits the user to enter text into the application, the default use is in recipes searches.
It is used by adding a class field to an input element.
